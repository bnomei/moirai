## Context

Exact-ID query collectors currently preserve every requested ID, so a duplicate live ID is yielded
and may be mutated multiple times. Reject duplicate identities during checked query preparation,
after contextual owner/stale validation and before any iterator, EntityRef, cache, or mutable closure
is exposed. Keep deterministic caller order for unique exact IDs.

## DoD

- Immutable, entity-only, cached/result-cached, and mutable exact-ID paths reject duplicates with a
  contextual `QueryError`.
- No callback or partial cursor progress occurs on rejection.
- Unique exact IDs retain caller order and existing stale/foreign diagnostics.

## Validation

Run all query suites, all-feature tests, strict Clippy, and Rust 1.75 no-default check. A fresh
Sol/high validator must inspect every exact-ID entry path and mutable alias prevention.

## Escalate If

One exact-ID path cannot share checked preparation without a public behavior change beyond duplicate
rejection.
