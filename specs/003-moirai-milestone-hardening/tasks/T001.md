## Context

`World::resource_scope` currently takes the registered resource and marks its store slot scoped,
then restores both only on normal return. An unwind therefore drops the value and permanently leaves
the slot inaccessible. Add an internal safe restoration guard that owns the taken value and enough
store state to restore or cancel the scope from `Drop`, without unsafe code or a std-only contract.
Normal `Result` returns retain the current added/changed tick behavior.

## DoD

- A caught callback panic leaves the resource value present and the scope sentinel cleared.
- A missing resource also clears the sentinel after unwind.
- The resource is dropped exactly once across normal return, replacement/removal, and unwind.
- Nested/locked resource error behavior remains unchanged.

## Validation

Run focused resource/hostile tests, all-feature tests, strict Clippy, and Rust 1.75 no-default check.
A fresh Sol/high validator must inspect unwind ownership and exact-drop behavior before completion.

## Escalate If

Safe restoration requires unsafe aliasing or changing the public callback signature.
