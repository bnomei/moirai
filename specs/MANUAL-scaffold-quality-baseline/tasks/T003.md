## Context

Treat absence of accidental public API as the Phase 1 behavior contract. Add downstream boundary
tests proving representative storage, registry, queue, runner, future root/prelude, and future
extension-trait paths are unavailable. README must accurately state that no library behavior is
implemented yet. Add representative `compile_fail` rustdoc examples in `src/lib.rs`; do not add a
separate compile-fail framework or a fake quickstart.

Do not claim a runnable ECS application before later phases provide real behavior.

## DoD

- Premature root, namespace, prelude, and extension-trait paths fail downstream compilation.
- README is accurate for the Phase 1 no-runtime surface and contains no fake quickstart.
- Representative private implementation paths are rejected downstream.

## Validation

```sh
cargo test --no-default-features
cargo test --features std
cargo test --features testkit
cargo test --doc --no-default-features
cargo doc --no-deps --all-features
```

## Escalate If

Privacy proof cannot be expressed as rustdoc `compile_fail`, or preserving README truth would require
a behavioral example backed only by a placeholder method or later-phase invariant.
