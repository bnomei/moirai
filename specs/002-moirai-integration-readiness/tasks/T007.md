## Context

Add `World::query_ids(spec, params) -> QueryIds` yielding `EntityId` and
`World::query_entities(spec, params) -> QueryEntities` yielding immutable `EntityRef`. An empty spec
enumerates every live entity; tag-only and exact-ID specs require no typed primary component.
`EntityRef` exposes `id`, checked `has<T>`, and checked immutable `get<T>` only. It must not permit
mutation or outlive its World borrow.

Extend `QuerySpec` with checked `ComponentId` selectors `with_id`, `without_id`, `with_tag_id`,
`without_tag_id`, `added_id`, and `changed_id`; retain the typed builders. Store multiple added or
changed selectors and match each group with OR semantics. Combining any added group with any changed
group remains `QueryError::ConflictingFilters`. Validate ComponentId owner, generation, registration,
tag kind, overlaps, exact-ID policy, and stale/foreign entity handling before iteration.

Entity queries participate in `QueryParams`, partial-iteration cursor commit rules, exact-ID policy,
`QueryCache`, and `QueryResultCache`. Add entity-spec cache builders and
`QueryCursor::{for_entities_from_start, for_entities_from_now, fork}`. Make raw fingerprint-based
cursor constructors crate-private. Do not add optional projection wrappers, Query3+, arbitrary
mutable tuples, name-based identity, raw component indices, or a Bevy query DSL.

`EntityId` must carry opaque World ownership in addition to slot and generation so bit-identical
allocator positions from different Worlds cannot alias. Thread the owner through allocation,
reservation, reconstruction, liveness, structural changes, and diagnostics. Preserve `Copy`, total
ordering, hashing, and the absence of public raw constructors; a representation-size change is
allowed. Deferred empty-entity spawn/despawn must advance query topology so membership and result
caches cannot remain valid across structural entity-set changes.

## DoD

- Empty, tag-only, dynamic-ID, exact-ID, cached, cursor, and partial entity queries work across table
  and sparse entities in deterministic order.
- EntityRef enables optional immutable reads without selecting a primary component.
- Added/changed OR groups use the captured query window and preserve cursor exhaustion semantics.
- Wrong-owner/stale ComponentId, EntityId, cursor, and caches fail contextually.
- Entity identity is owner-bearing across all World and allocator paths, including reserved and
  reconstructed handles; cross-World slot/generation collisions are rejected.
- Deferred spawn/despawn of entities without components invalidates structural query caches.
- Existing Query1/Query2 and mutable closure APIs remain source-compatible except for T001's removal
  of unnecessary Clone bounds.

cursor invariants, consumer needs prove Query3+/mutable dynamic projection is unavoidable,
