## Context

Remove the universal `Clone` requirement from component registration, table/sparse storage, bundles,
World component access, typed queries, and the public `Commands` insertion/removal surface. Prepare
the erased command representation to hold non-Clone owned values without applying them by clone;
T002 owns the consuming deferred-application seam and new bundle/query command operations. Erased
table columns must transfer owned values and their stored change ticks between archetypes. Structural
insert/remove must never clone retained values; removal returns the original owned value. Preserve
checked registration/storage-kind behavior, archetype rollback, owner checks, query borrowing,
no-std support, and `forbid(unsafe_code)`.

Do not change event payload bounds in this task. Do not add unsafe erased moves, proc macros, a new
crate, raw entity access, or downstream adapters.

## DoD

- A component with no `Clone` implementation can be registered, spawned, queried, mutated,
  structurally migrated, removed, queued through the existing command API, and dropped.
- Adding/removing an unrelated component preserves the retained component's identity and change
  ticks without invoking its clone operation.
- Success and rollback paths drop every owned value exactly once and leak none.
- Existing table, sparse, bundle, query, owner, and failure contracts remain green.

Add focused non-`Clone`, clone-count, drop-count, identity, tick-transfer, and failed-migration

On `needs_fix`, create a repair packet and rerun focused validation plus a new Sol/high review; use

## Escalate If

Safe owned transfer cannot be implemented without unsafe code, storage representation must become
public, preserving ticks requires changing their documented temporal semantics, or removing Clone
would require an unplanned query/API redesign.
