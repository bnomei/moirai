## Context

Build on T001's owned erased-value transfer. Deferred values must be consumed when applied rather
than borrowed and cloned. Add atomic `Commands::insert_bundle(entity, bundle)` for an existing live
entity. Extend `QueryCommands` with typed `insert`, `remove`, and `insert_bundle` operations so
systems can request structural changes while iterating without collecting IDs and reacquiring the
World. Report structural-command rejection as `QueryError::CommandRejected`.

Bundle validation/enqueue is transactional: if bundle writing, registration, owner/liveness
checking, or enqueue fails, the pending batch contains no prefix of that bundle and all values are
returned/dropped exactly once. Application at a later flush keeps the existing batch failure and
atomicity rules.

## DoD

- Erased deferred insert owns and consumes a non-`Clone` value exactly once.
- Commands can atomically apply a dynamic/tuple bundle to an existing entity.
- Query effects can insert, remove, and insert bundles while iteration remains borrow-safe.
- Rejected bundle enqueue leaves the batch unchanged and exposes the dedicated query error.
- Existing spawn/despawn ordering, flush, rollback, and wrong-owner guarantees remain green.

## Validation

Add focused non-`Clone`, existing-entity bundle, query-effect, partial-bundle rejection, and
drop-count tests, then run:

```sh
cargo test --test commands_deferred --test query_effects --test phase3_failure_contracts
cargo test --all-features
cargo clippy --all-targets --all-features -- -D warnings
cargo +1.75 check --lib --no-default-features
```

Orchid must run a fresh Sol/high validator, distinct from the worker, against the canonical worker
report. The validator must inspect the diff and `git-touched`, reproduce focused tests, and review
ownership, transaction rollback, flush failure, query borrowing, and public error semantics. On
`needs_fix`, create a repair packet and rerun focused validation plus a new Sol/high review; use
Terra/medium only for a bounded local defect with no contract choice. Stop after three repair
rounds, the same root defect twice, a blocked verdict, attribution conflict, or a required
architecture decision outside this spec. Do not stage or commit before `pass`.

## Escalate If

Atomic enqueue requires a public transaction type, rollback cannot preserve exactly-once drops,
query-scoped commands require relaxing World aliasing, or the task reveals a need for arbitrary
mutable query tuples.
