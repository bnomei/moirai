## Context

Add public non-exhaustive `EntityScratchError` with `WrongWorld`, `StaleEntity { entity }`, and
`EntityNotLive { entity }`. Add public `EntityScratch<V>` as transient owner-bound storage keyed by
full generational `EntityId`, with these contracts:

```rust
pub fn new(world: &World) -> EntityScratch<V>;
pub fn insert(&mut self, world: &World, entity: EntityId, value: V)
    -> Result<Option<V>, EntityScratchError>;
pub fn get<'a>(&'a self, world: &World, entity: EntityId)
    -> Result<Option<&'a V>, EntityScratchError>;
pub fn get_mut<'a>(&'a mut self, world: &World, entity: EntityId)
    -> Result<Option<&'a mut V>, EntityScratchError>;
pub fn remove(&mut self, world: &World, entity: EntityId)
    -> Result<Option<V>, EntityScratchError>;
pub fn retain_live(&mut self, world: &World) -> Result<usize, EntityScratchError>;
pub fn clear(&mut self);
pub fn len(&self) -> usize;
pub fn is_empty(&self) -> bool;
```

`insert` returns the replaced value; `get`/`get_mut`/`remove` return `Ok(None)` only for a live
entity with no scratch entry. Entity-facing operations first require the World used at construction,
then validate generation/liveness and never fall through to an entity that reused the same slot.
`retain_live` first validates the World owner, removes all entries no longer live at their recorded
generation, and returns the removed count. Existing `EntityId` values are explicitly world-relative
and do not carry an owner token; `WrongWorld` therefore validates the supplied World against the
scratch owner rather than claiming to identify the provenance of a coincidentally equal raw handle.

Implement with safe `alloc` collections. Keep slot packing, WorldOwner, and allocator internals
private. Do not add raw-index conversion, serialization/persistence traits, global storage, or
automatic attachment to World; this is system-local/captured scratch state.

## DoD

- Scratch values round-trip for live originating entities.
- A foreign World argument is rejected, and despawned/reused generations cannot read or mutate prior
  values in the bound World.
- `retain_live` deterministically cleans stale entries and preserves live generations.
- Value replacement/removal/clear/drop obey exactly-once ownership.
- The type is available from the curated root and prelude without exposing owner or slot internals.
