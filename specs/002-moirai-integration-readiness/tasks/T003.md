## Context

Standardize registered events on a typed broadcast contract of `E: Clone + 'static`. Readers own
copied payloads and remain independent. Sending must not compact away prior frame events merely
because no reader has started; a frame queue clears only when its registered Update or Render
operation completes. Preserve bounded/manual retention, lag reporting, reader start policy,
component lifecycle channels, allocation reuse, wrong-owner checks, and idle host access.

Do not implement system event-role metadata here; T004 owns role compilation and runtime access.

## DoD

- Two or more frame events sent before the first consumer reader are all observed in order.
- Independent readers observe the complete broadcast without moving payloads away from each other.
- Update and Render frame channels clear only at their respective operation boundary.
- Manual/bounded compaction and lag behavior remain deterministic and documented by tests.
- Public send/read/registration bounds consistently express `Clone + 'static`.

## Validation

Add first-reader, multi-send, independent-reader, Update/Render boundary, manual/bounded, lifecycle,
and payload-drop regressions, then run:

```sh
cargo test --test events --test component_lifecycle --test allocation
cargo test --all-features
cargo clippy --all-targets --all-features -- -D warnings
cargo +1.75 check --lib --no-default-features
```

Orchid must run a fresh Sol/high validator, distinct from the worker, against the canonical worker
report. The validator must inspect the diff and `git-touched`, reproduce focused tests, and review
queue lifetime, compaction, lag, operation clearing, payload ownership, and API-bound consistency.
On `needs_fix`, create a repair packet and rerun focused validation plus a new Sol/high review; use
Terra/medium only for a bounded local defect with no contract choice. Stop after three repair
rounds, the same root defect twice, a blocked verdict, attribution conflict, or a required
architecture decision outside this spec. Do not stage or commit before `pass`.

## Escalate If

Correct retention requires unbounded lifetime beyond the registered policy, independent readers
cannot be preserved under the explicit Clone contract, or operation clearing requires schedule-role
behavior assigned to T004.
