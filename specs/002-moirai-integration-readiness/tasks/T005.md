## Context

Publish `Condition::from_world(F)` for cloneable no-std predicates captured through the existing
`Rc` representation. Add `ScheduleBuilder::set_stage_flush_mode(label, mode)`: Update stages accept
`Final` or `Stage`; Render stages accept only `Final`. A system descriptor accepts `Final` or
`AfterSystem`; reject `Stage` on a system and reject `AfterSystem` on a stage rather than silently
ignoring either. `add_stage` retains its established defaults.

Remove builder call-order sensitivity: adding a FixedUpdate system before `fixed(config)` is valid,
and final `build` alone rejects a fixed system without config or a config without that stage.

Add `System::before_set(&SystemSet)` and `System::after_set(&SystemSet)`, plus
`ScheduleBuilder::order_set_before(&SystemSet, &SystemSet)` and `order_set_after`. Expand these
relations into deterministic system edges before topological sort. Unknown/empty sets retain normal
registration semantics; any expanded edge crossing stage boundaries is a build error. Preserve one
set membership per system and existing string system edges.

## DoD

- Public world predicates can gate systems and sets using read-only resource fields.
- Stage/system flush placement is explicit, effective, and validated; Render stays topology-only.
- Fixed configuration is independent of authoring call order.
- System-to-set and set-to-set edges affect deterministic ordering and cycle detection.
- Cross-stage expanded ordering fails at build with contextual labels.

## Validation

Add custom predicate, flush visibility/error, fixed call-order, set expansion/cycle, and cross-stage
rejection tests, then run:

```sh
cargo test --test schedule --test schedule_observer --test world_lifecycle_state --test public_api --test prelude
cargo test --all-features
cargo clippy --all-targets --all-features -- -D warnings
cargo +1.75 check --lib --no-default-features
```

Orchid must run a fresh Sol/high validator, distinct from the worker, against the canonical worker
report. The validator must inspect the diff and `git-touched`, reproduce focused tests, and review
predicate aliasing, no-std captures, flush visibility, Render authority, graph expansion,
determinism, cycles, and builder compatibility. On `needs_fix`, create a repair packet and rerun
focused validation plus a new Sol/high review; use Terra/medium only for a bounded local defect with
no contract choice. Stop after three repair rounds, the same root defect twice, a blocked verdict,
attribution conflict, or a required architecture decision outside this spec. Do not stage or commit
before `pass`.

## Escalate If

Set expansion needs multiple set membership, a world predicate would permit mutation or non-static
borrows, existing stage defaults must change, or cross-stage ordering cannot be rejected without
changing the operation execution model.
