## Context

Add fluent typed metadata `System::emits::<E>()`, `System::consumes::<E>()`,
`System::consumes_on_add::<T>()`, and `System::consumes_on_remove::<T>()`. Compile roles into system
metadata. Build must reject an unregistered channel, an event whose registered Update/Render owner
does not match the system stage operation, a consumer without an external source or declared
producer, and a producer that is not ordered before its consumer through the compiled graph.
Lifecycle consumption resolves through the component's registered add/remove channel.

Registered component-added and component-removed channels are intrinsic Update-operation producers:
`consumes_on_add`/`consumes_on_remove` satisfy producer existence without `external_source()` or a
system `.emits` declaration, and they are exempt from system-producer reachability. Their visibility
is determined by the existing structural flush boundary, so a lifecycle consumer sees only events
made visible before it runs. A lifecycle role attached to a Render system is a wrong-operation build
error.

`EventOptions::external_source()` exempts producer-existence/order only; registration and operation
ownership still apply. While a system body or its `QueryEffects` executes, send/read/reader creation
must match the current system's compiled role. Idle host access outside a run remains unrestricted.
Role enforcement must use typed IDs/metadata, not event names.

## DoD

- All four role builders compile into immutable system metadata.
- Missing registration, wrong operation, missing producer, and unreachable producer ordering are
  distinct deterministic build failures.
- External-source channels build without a system producer but still enforce owner operation.
- Lifecycle consumers build against intrinsic Update producers and require no fake emitting system.
- Runtime undeclared send/read from ordinary systems and query effects is rejected without corrupting
  event reader or command state.
- Existing resource/access, cycle, stage, lifecycle, and event behavior remains green.

not stage or commit before `pass`.

## Escalate If

Correct producer reachability conflicts with existing deterministic topological ordering, runtime
role checking requires unsafe/global state, lifecycle channels cannot be resolved from registered
component IDs, or host access would need to be restricted outside schedule execution.
