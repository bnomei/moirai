## Context

Add fluent `WorldBuilder::insert_resource(value)` and `insert_state(initial)`, with matching
`AppBuilder` forwarding methods. Each call both registers and seeds its type. Seeds are installed
before schedule validation; repeated seeds of the same type are last-call-wins and produce one
initial added/changed tick, not a sequence of runtime mutations. Existing register-only APIs remain.

Introduce public non-exhaustive `StateError` for conflicting pending destinations. A request equal
to the current state is a no-op when no different transition is pending; repeating the same pending
destination is also a no-op. `state::apply::<S>` must declare `requires_resource::<State<S>>()`, so
missing state is rejected during schedule construction. Add `App::fault() -> Option<&AppFault>` and
retain the first terminal fault through later calls.

## DoD

- AppBuilder can seed every resource/state required by its systems before schedule validation.
- Last-call-wins seeds have stable initial added/changed tick semantics.
- Same-state and duplicate pending requests are idempotent; conflicting pending requests return
  `StateError` rather than an unrelated World storage error.
- The state apply system declares its resource requirement.
- `App::fault()` exposes the original terminal fault after execution, tick, or panic failure.

## Validation

Add builder-order, duplicate-seed/tick, missing-state build, transition truth table, and first-fault
retention tests, then run:

```sh
cargo test --test resources --test world_lifecycle_state --test error_display --test public_api --test prelude
cargo test --all-features
cargo clippy --all-targets --all-features -- -D warnings
cargo +1.75 check --lib --no-default-features
```

Orchid must run a fresh Sol/high validator, distinct from the worker, against the canonical worker
report. The validator must inspect the diff and `git-touched`, reproduce focused tests, and review
seed ordering/ticks/drop behavior, state transition truth tables, build-time requirements, error
compatibility, and terminal fault retention. On `needs_fix`, create a repair packet and rerun focused
validation plus a new Sol/high review; use Terra/medium only for a bounded local defect with no
contract choice. Stop after three repair rounds, the same root defect twice, a blocked verdict,
attribution conflict, or a required architecture decision outside this spec. Do not stage or commit
before `pass`.

## Escalate If

Builder seeds require changing World's initial tick epoch, last-call-wins cannot avoid observable
intermediate drops, StateError would break an established public conversion relied on by tests, or
preserving the first panic fault conflicts with rethrow behavior.
