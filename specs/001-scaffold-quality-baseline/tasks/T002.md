## Context

Create the final private physical module layout without exposing allocator, registry, sparse/table
storage, raw commands, event queues, query plans, or schedule runner modules. Phase 1 publishes no
semantic namespace, root re-export, prelude, or extension trait. The documented root list is the
final target, not a list of stubs to emit now.

Do not declare future-owned public structs/enums or traits. In particular, `Bundle`/`BundleWriter`
and `diagnostics::Observer` arrive with their real Phase 3/4 contracts; no empty trait, guessed
method, constructor, default, panic, no-op, or invented runtime behavior is allowed.

## DoD

- Private implementation children cannot be named as public API.
- No semantic namespace, root re-export, prelude, or extension trait leaks before its owner phase.
- Crate docs retain the frozen final root/prelude policy without presenting it as live Phase 1 API.

## Validation

```sh
cargo check --no-default-features
cargo doc --no-deps --all-features
```

Request an independent Sol/high review of visibility and the no-placeholder rule before completion.

## Escalate If

The private physical layout cannot be created without leaking a public path, or the documented
final-path contract would need an architecture amendment.
